# Contributing

Thanks for your interest in this project.

This repository is part of the **SudoCode Pentesting Methodology Guide**
series maintained by SudoChef. It is primarily a solo-maintained
educational reference.

## How to Contribute

At this time, contributions are limited to:

- **Corrections** — broken syntax, outdated operators, factual errors
- **Additions** — operators, dorks, or tools that are missing and verifiable
- **Clarity improvements** — better plain-English explanations

## How to Submit

1. Fork the repository
2. Create a new branch — `fix/your-description` or `add/your-description`
3. Make your changes
4. Open a Pull Request with a clear description of what changed and why

## What Will Not Be Accepted

- Content that enables targeted harassment or illegal activity
- Dorks designed to expose private individuals' personal information
- Anything that violates GitHub's Terms of Service

## Standards

- Plain English first, technical second
- Real working examples only — no made-up syntax
- Follow the existing file structure and signature format
